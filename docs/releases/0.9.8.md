## Summary
- Avoid an unconditional destination existence check in the normal copy and move hot path.
- Make `--prune-empty-dirs` also remove subdirectories that were already empty before processing started.
- Add integration coverage for both behaviors.

## Changes
- Limit the pre-check for existing destination files to `--dry-run` runs, letting normal copy and move operations use the actual file operation result.
- Expand the deferred prune pass to consider all source subdirectories, not just directories touched by removed files.
- Add integration tests for dry-run with an existing target and for pruning pre-existing empty subdirectories.

## Verification
- `dotnet build move.slnx`
- `dotnet test move.slnx`
